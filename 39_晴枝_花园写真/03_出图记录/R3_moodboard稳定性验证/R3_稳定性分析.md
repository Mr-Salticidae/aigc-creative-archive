# R3 moodboard 稳定性分析

## 输入

- 来源压缩包：`midjourney_session (58).zip`
- 解包目录：`03_出图记录/R3_moodboard稳定性验证/`
- 数量：12 张，3 组，每组 4 张
- 联系表：`_R3_contact_sheet.png`

## 总判断

R3 已经证明：同一 prompt + moodboard 的组合稳定有效。

这轮不是偶然命中。12 张里大多数都稳定保留了晴枝的核心视觉：黑发、东方面孔、花叶前景、叶影、春末初夏光感、轻微笑意或安静凝视。moodboard 确实补上了纯 prompt 难以稳定获得的账号审美底色。

## 稳定成立的维度

1. **面孔谱系稳定**
   - 大多数图都有相近的鹅蛋脸、杏仁眼、黑发、清透肤色。
   - 比 R1 纯 prompt 更接近“檐下”账号审美，但整体更暖、更春天。

2. **春日光影稳定**
   - 叶影、花前景、散射日光重复出现。
   - 氛围稳定在春末初夏，没有明显滑回冬感。

3. **发丝质感稳定**
   - 乱发和风感是本轮最大加分项之一。
   - 发丝参与面部遮挡，增加真实摄影瞬间感。

4. **角色安全边界稳定**
   - 整体没有幼态化，也没有明显性感化。
   - 人物仍是成年女性写真方向。

## 仍需控制的风险

1. **笑意过甜**
   - 少数图会滑向亲和甜妹，角色记忆点下降。

2. **春日过绿**
   - 个别图绿色环境压得较重，容易变成普通户外人像。

3. **檐下暗面回流**
   - 少数图如 G3-3 很有张力，但冷感和凝视强度偏回“檐下暗面”，适合作为情绪备选，不适合作为晴枝主基准。

4. **花只是环境**
   - 有些图的花叶是漂亮前景，但还没有成为晴枝的明确视觉签名。

## 方法结论

晴枝项目正式确认采用四层金字塔叠用：

1. **账号级锚点**：personalize / moodboard，锁账号审美底盘。
2. **角色级锚点**：晴枝角色卡，锁人物和气质边界。
3. **图像级锚点**：R2-0 作为核心锚点；G2-2 作为技术备用 `oref` 候选。
4. **单图参数**：`--v 8.1 --style raw --ar 3:4 --no ...`，锁具体镜头和安全边界。

## 下一步

进入 `oref` 扩展测试。

建议先用 R2-0 做核心锚点测试，必要时用 G2-2 做技术备用对照，测试 3 个半身 / 中景场景：

1. 花园小径迎风走来
2. 手指掠过花枝的半身侧脸
3. 树荫下回头看向镜头外

`--ow` 建议从 100 开始；如果脸漂，升到 200；如果服装和背景被参考图锁死，降到 50。
