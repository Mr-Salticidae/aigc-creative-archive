# R1 面部特写诊断

## 输入图

- R1 最佳图：`R1_best_晴枝_face.png`
- 爆款参考：`../参考图/檐下爆款_face_reference.png`

## 当前图成立的地方

- 春末初夏氛围成立：阳光、叶影、前景虚化都自然。
- 面部干净，皮肤和光线舒服，有写真感。
- 不幼态、不性感化，安全边界和角色年龄感基本成立。

## 当前图不够惊艳的原因

1. 面孔太顺滑，缺少“骨相记忆点”。脸型、颧骨、下颌线都偏柔，第一眼好看，但不容易被记住。
2. 眼神太亲和。图一像一个漂亮的人在花园里微笑，图二像一个角色正在看见你，视线有压迫力和故事密度。
3. 视觉签名太弱。图二的白花、枝条、湿发、檐下结构都在帮人物建立 IP 识别；图一的花园元素只是环境。
4. 构图太舒适。图一是常规美图逻辑，图二有遮挡、仰视、枝条切割、冷暖反差，画面更有张力。
5. 情绪太满。图一已经把“轻微笑意”给出来了，留白少；图二的表情更克制，观众会想补完她的故事。

## 可迁移规则

- 迁移“檐下”的鹅蛋脸、杏仁丹凤眼、清晰骨相、干净面部留白。
- 迁移“檐下”的遮挡层次：枝条 / 花 / 发丝 / 前景虚化压到脸前。
- 迁移“檐下”的视线强度：不是甜笑，而是安静地看向镜头。
- 迁移“檐下”的视觉签名机制：花必须和头发、脸、构图发生关系，不能只是手里道具。

## 不迁移的部分

- 不迁移冬感、雪、古风身份、玉兰、病弱冷白。
- 不迁移悲伤氛围。
- 不让晴枝变成檐下本人。

## 下一轮方向

下一轮先做“惊艳面孔锁定”：更近的脸部特写，轻微低角度或枝叶遮挡，眼神更稳，笑意更少，花与发丝建立视觉签名。
