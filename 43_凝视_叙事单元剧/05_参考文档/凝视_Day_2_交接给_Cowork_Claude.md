## 四、关键工具栈(v2 重大变更)

### ⚠️ 主静态图工具:Niji V7 → Nano(LibLib 画布工作流)

**变更原因**:跳蛛先生在 v1→v2 过程中通过 LibLib 平台的画布功能,验证了 Nano(Gemini 2.5 Flash Image)在"画布三件套"输入模式下,**画风一致性 + 角色一致性都优于 Niji V7,且出图成功率 2-3 轮就能拿满意成品**(Niji 通常要 5-10 轮抽卡)。

**画布三件套配方**(已批量验证有效):

```
画布输入 1:角色 Day 1 主图(作为画风种子 + 角色身份种子)
画布输入 2:角色关键元素清单(角色卡里有,如 Veritia 的:
            玫瑰金长发 / 天蓝色眼睛 / 破碎金王冠 + 红宝石 /
            单滴泪痕 / 苍白手指 + 暗金宝石戒指 / 银色指甲 /
            暗红丝绒袍 / 末代王朝氛围)
画布输入 3:角色视觉特征描述(角色卡里的中文散文段落)
简化提示词:[要做什么,如"三视图""全身照""九宫格表情""王冠破碎特写"]
```

**重要发现**:Nano 在纯文本 prompt 下确实是"清水插画"风格,**不能做 Day 1 风格延续**——这条规则没变。但通过画布三件套,**Nano 能学到 Day 1 主图的 painterly + chiaroscuro 画风**,产出和 Day 1 高度一致的成品。

- ❌ 出现明显鼓点 → 重做

## 九、必须传承的方法论原则

### "蒙眼剪辑法"(核心工作分工)

LLM 看不见视频 → 创作者(跳蛛先生)用人耳听 BGM 标卡点 → 反馈给 LLM → LLM 用代码精确执行

### 严格分工

- **创作决策**(图文/风格/BGM/卡点/排序):**全部交给跳蛛先生**
- **工程实现**(代码/特效/渲染):**全部交给你**

### 反馈精度决定迭代速度

✅ 好反馈:"Veritia 出现得晚了 200ms" / "色差转场太弱"
❌ 坏反馈:"再调一下" / "感觉不对"

要在合作中**主动训练**跳蛛先生提供精度反馈,而不是含糊反馈。

### Niji V7 三层结构(角色卡里的 prompt 结构,仍然有效作为参考)

每个角色的 Niji V7 prompt 拆成 3 层:
- **Layer 1 - 角色 ID**(锁住"她是谁"):角色独有的身份特征,逐字保留
- **Layer 2 - 画风技法**(锁住"她长什么样"):painterly / chiaroscuro / dust particles 等共享词汇,逐字保留
- **Layer 3 - 构图变量**(可改):镜头景别、姿态、特定瞬间

虽然主工具切到 Nano,但这套方法论仍是理解 Day 1 画风基因的钥匙。

见第四节,与 Niji 三层结构并存,但 Nano 路线是默认。

## 十、必须避开的坑(v2 更新版)

1. **不要用 GPT Image 2 / Nano 做 Day 1 风格延续——前提是纯文本 prompt**。
   Nano + LibLib 画布三件套**可以**做 Day 1 风格延续,这条规则在画布工作流下不适用。
2. **不要假设 Niji V7 有 cref/oref**——它没有,角色一致性靠 Layer 1+2 逐字保留 + seed。
3. **三视图不要喂给视频生成模型**——会让模型把"白底 T-pose"学成角色特征。
   三视图(正/侧/背 T-pose 黑底/白底)→ 角色档案 / 手办原型 / 3D 建模
   Seedance 参考图(生活化场景、3/4 角度、动态姿势)→ 视频生成
4. **不要先生成 BGM 再剪辑**——必须先 BGM 后剪辑,否则音画一定不合。
