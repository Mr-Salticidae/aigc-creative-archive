# 《晴枝》项目档案 v0.1

## 一句话命题

一个成年东亚年轻女性在春末初夏的花园中漫步，明亮、松弛、清醒、有生命力，像刚刚重新喜欢上这个世界。

## 从“檐下”继承什么

本项目不继承“檐下”的清冷古风身份，也不混用“玉兰”视觉签名。

继承的是方法：

- 先建立角色卡，再写故事和 prompt
- 先锁人物气质，再换场景
- 让每张图都能回答“她是谁”
- 把视觉签名设计成轻量锚点，而不是硬贴符号

## 与此前悲伤氛围作品的区别

旧方向常用：冷、孤、夜、雨、沉郁、失落、等待。

晴枝方向改用：风、晨光、花影、草木、轻微笑意、恢复、自由感。

情绪不是“没有故事的快乐”，而是“从低气压里走出来后的身体轻盈”。

## 创作风险

- 过甜：容易变成普通平台花园甜妹图
- 过幼：容易损失气质和安全边界
- 过网红：姿态、妆容、服装一旦太商业，会丢失故事感
- 过梦幻：童话滤镜太重，会削弱真实生活切片

## 当前测试目标

第一轮只验证一件事：晴枝能否在“花园小径迎风走来”的主视觉里成立。

判断标准：

- 人物是否成年、自然、不幼态
- 元气感是否来自光、风、动作和眼神，而不是硬笑
- 画面是否有春末初夏的空气感
- 是否保留轻故事感，而不是普通摆拍

## 2026-06-07 方法修正：必须启用账号级锚点

R1 纯 prompt 追脸的问题是：画面漂亮，但面孔惊艳度和账号 IP 审美底色不足。

R2 使用修改前的面部特写 prompt，并加入 moodboard 后，面孔质感、五官比例、发丝、叶影和整体审美显著提升。由此确认：晴枝项目不能只靠 prompt 修五官，必须回到《审美锚点 02》的角色一致性金字塔。

本项目后续默认叠用：

1. 账号级锚点：personalize / moodboard，用于锁定跳蛛先生账号审美底色。
2. 角色级锚点：角色卡 + 文字锁定段，用于定义“晴枝不是檐下本人”。
3. 图像级锚点：选出 R2 最佳图后，用 `--oref` 做脸部一致性扩展。
4. 单图参数：`--ar`, `--v 8.1`, `--style raw`, `--no` 等。

注意：R2 成功图适合进入 oref 候选，不适合直接作为 sref 锚点。sref 应保持风格签名纯净，避免主体污染。
