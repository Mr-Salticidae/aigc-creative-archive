# 仓库结构说明

## 目标

本仓库不是单个作品仓库，而是 AIGC 创作总归档。每个项目作为一个独立子目录存在，方便跨设备同步、检索和延续创作。

## 项目目录命名

沿用本地工作站编号：

```text
NN_项目名/
```

示例：

```text
39_晴枝_花园写真/
```

## 推荐项目结构

```text
NN_项目名/
├── README.md
├── 00_项目档案.md
├── 01_角色卡/
├── 02_MJ_prompts/
├── 03_出图记录/
├── 04_精选成图/
└── 05_发布归档/
```

## 各目录职责

| 目录 | 内容 |
|---|---|
| `README.md` | 项目简介、当前状态、下一步 |
| `00_项目档案.md` | 创作判断、方法修正、关键结论 |
| `01_角色卡/` | 人物设定、视觉锚点、禁忌、角色锁定段 |
| `02_MJ_prompts/` | Midjourney prompt、参数说明、迭代版本 |
| `03_出图记录/` | 每轮出图、联系表、诊断、筛选记录 |
| `04_精选成图/` | 候选基准图、发布候选、精选版本 |
| `05_发布归档/` | 发布文案、标题、平台适配、最终归档 |

## 提交建议

优先提交：

- Markdown 文档
- 角色卡
- prompt
- 诊断和复盘
- 联系表
- 候选基准图
- 精选成图

谨慎提交：

- 大批量原始出图
- 视频素材
- 压缩包
- 临时导出文件

默认不提交：

- 下载源 zip
- 临时缓存
- 可重复生成的大型中间文件

如果某个项目开始产生大量视频或超大图像素材，再单独评估是否启用 Git LFS。
