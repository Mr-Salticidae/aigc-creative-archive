# 《异常值》立项书

- **立项日期**：2026-06-13
- **类型**：概念推演型 AI 短片 / 哲学科幻 / 系统档案影像
- **状态**：风格基底验证阶段

---

## 一、核心命题

如果一个系统能根据行为数据生成你未来一万种人生，那么你现在的选择，究竟是自由意志，还是概率最高的一条路径？

**核心句**：自由意志，也许只是系统暂时无法压缩的异常值。

## 二、叙事原则

不追踪固定主角，从**系统视角**观看人类。人、脸、年龄、场景都可变，但所有画面服务同一命题：**系统可以预测人的惯性，却无法完全压缩人的偏离。**

四种一致性（按优先级）：
1. 系统视角一致——所有画面像系统档案/人生预测/数据模拟
2. 命题一致——所有镜头服务"预测人生 / 偏离路径 / 异常值"
3. 情绪一致——冷静、压迫、克制、锋利
4. 视觉一致——黑蓝背景、暖黄文字、灰白人物、暗红异常值

> **角色一致性不进优先级**。群像/命题叙事天然不需要锁脸——这是本片对 AI 制作的最大顺风点。

## 三、为什么这部片适合 AI

| AI 技术短板 | 本片的天然规避 |
|---|---|
| 角色锁脸 / 漂移 | 不固定主角，群像、残影化、背身/不露脸 |
| 写实人脸特写恐怖谷 | 人物灰白低饱和、远观、档案式小图 |
| 复杂肢体动作 | 剧本明确"不要过度动作化"，静观为主 |
| 场景跳切穿帮 | 系统视角 + 黑场/字卡天然合法 |

## 四、A/B 双轨制作模型

- **A 系统层**（系统文字、分支树、方块溶解、压缩失败）→ Remotion 代码生成，像素级可控，契合"旧终端档案"美学。
- **B 截帧层**（人生瞬间、群像）→ MJ 静图 + sref 锁风格 → 轻量 i2v。

两层锚点分离：A 锚 = 方块溶解母题（暗调）；B 锚 = 截帧②（深夜刷招聘）。

## 五、识别出的两个真正难点

1. **节奏单调风险**（最大坑）：六章皆"悬浮截帧 + 冷旁白"，结构同质。靠尺度跃升的呼吸感与视觉密度变化留人，而非剧情钩子。
2. **旁白质感**：旁白驱动型，TTS 的冷感、克制、停顿决定半部片成败。

## 六、制作顺序

```
① 风格基底测试（当前）→ ② 锁 sref + 调色 + 系统文字模板
→ ③ 六章分镜表 → ④ B 层批量截帧 + A 层 Remotion 各屏
→ ⑤ 旁白 TTS + 节奏打点 → ⑥ 剪辑合成（蒙眼剪辑法）
```

强制纪律：**先验证视觉语言，再铺量**。概念推演型最忌先生产几百张图、最后发现风格焊不住推倒重来。
